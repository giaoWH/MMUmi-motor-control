MMUmi-motor-control 是一个基于 DM3507 电机与 MC02 控制板开发的双电机力反馈遥操作嵌入式项目。项目核心功能是实现双电机的精准控制与力反馈数据交互，通过控制板 USB 接口完成电机数据的实时回传，同时具备上电初始化、前馈力矩可调等实用功能，适用于遥操作相关的嵌入式控制场景。
## 核心功能
- 双电机协同控制：基于 DM3507 电机，实现双电机同步运行与独立控制，适配遥操作场景下的动力输出需求。
- USB 数据回传：通过 MC02 控制板的 USB 接口，实时返回双电机的运行数据（如位置、转速、力矩等），便于上位机进行数据监测与后续处理。
- 上电位置初始化：设备上电后，自动将双电机当前位置设置为初始基准位置，简化操作流程，确保每次启动后控制基准一致。
- 前馈力矩可调：支持手动调节电机前馈力矩参数，可根据实际应用场景（如负载变化、遥操作灵敏度需求）灵活适配，提升控制精度与稳定性。
## 硬件说明
### 核心硬件组件
- 电机：DM3507 直流电机（双电机配置），具备动力强劲、控制精度高的特点，适配中小负载遥操作场景。
- 控制板：MC02 嵌入式控制板，作为项目核心控制单元，负责电机驱动、数据采集与 USB 通信。
- 通信接口：控制板集成 USB 接口，用于与上位机进行数据交互，传输电机运行数据与控制指令。
## 软件环境
- 开发语言：C 语言，适配嵌入式底层开发，兼顾运行效率与代码可读性。
- 开发工具：MDK-ARM，用于项目编译、调试与下载，适配 MC02 控制板的开发流程。
- 依赖库：STM32_USB_Device_Library（位于 Middlewares 目录），用于实现 USB 接口的数据传输功能。
## 项目结构
项目目录结构遵循嵌入式工程规范，各核心目录功能如下：
- Core：核心代码目录，包含主程序、电机控制逻辑、数据处理等核心功能模块。
- Drivers：驱动目录，包含 DM3507 电机驱动、MC02 控制板外设驱动等。
- MDK-ARM：MDK 工程目录，包含项目编译配置、调试配置等文件，可直接用于开发调试。
- USB_DEVICE：USB 通信相关代码目录，负责 USB 接口的初始化与数据收发。
- Middlewares/ST/STM32_USB_Device_Library：USB 通信依赖库，实现 USB 设备功能与数据传输。
- User：用户自定义代码目录，可根据实际需求添加自定义控制逻辑、参数配置等。
- CtrBoard.ioc：STM32 配置文件，用于配置控制板外设（如 GPIO、USB 等），可通过 STM32CubeMX 编辑。
## 使用说明
1. 硬件连接：将双 DM3507 电机正确连接至 MC02 控制板的电机接口，通过 USB 线将控制板与上位机连接。
2. 程序下载：通过 MDK-ARM 打开项目工程，编译无误后，将程序下载至 MC02 控制板。
3. 上电运行：控制板上电后，系统自动将双电机当前位置设置为初始位置，电机进入待机状态。
4. 参数调节：根据实际需求，调节电机前馈力矩参数，实现不同负载、不同灵敏度下的稳定控制。
5. 数据监测：通过上位机读取 USB 接口返回的双电机数据，实时监测电机运行状态。
## 注意事项
- 电机接线需正确，避免正负极接反导致电机或控制板损坏。
- 调节前馈力矩时，需逐步调试，避免力矩过大损坏电机或负载。
- 程序下载前，需确认 MDK-ARM 工程配置与 MC02 控制板型号匹配，避免编译或下载失败。
- USB 通信过程中，避免频繁插拔 USB 线，防止数据传输中断或控制板异常。
